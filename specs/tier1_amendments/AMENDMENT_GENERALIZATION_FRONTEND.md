## FILE 10: frontend/src/lib/utils.ts — formatDateIST / formatISTDate (RETROFIT — apply to the existing FRONTEND_SUPPLEMENT_01 implementation)

**A different category of generalization issue, found in this sweep — timezone/locale hardcoding, not a company name.** `formatDateIST` hardcodes `timeZone: 'Asia/Kolkata'` and locale `'en-IN'`. For a deployment outside India, every timestamp in the UI (audit trail, session history, PDF exports) would display in Indian time regardless of where the company or its employees actually are.

```typescript
// FIND:
//     export function formatDateIST(date: Date | string): string {
//       const d = typeof date === 'string' ? new Date(date) : date
//       return d.toLocaleString('en-IN', {
//         day: 'numeric', month: 'short', year: 'numeric',
//         hour: '2-digit', minute: '2-digit',
//         timeZone: 'Asia/Kolkata',
//       })
//     }
//
// REPLACE WITH:

const DEPLOY_LOCALE = process.env.NEXT_PUBLIC_DEPLOY_LOCALE || 'en-IN'
const DEPLOY_TIMEZONE = process.env.NEXT_PUBLIC_DEPLOY_TIMEZONE || 'Asia/Kolkata'

export function formatDateLocalized(date: Date | string): string {
  const d = typeof date === 'string' ? new Date(date) : date
  return d.toLocaleString(DEPLOY_LOCALE, {
    day: 'numeric', month: 'short', year: 'numeric',
    hour: '2-digit', minute: '2-digit',
    timeZone: DEPLOY_TIMEZONE,
  })
}
// Keep a deprecated alias so existing call sites are not all broken at once:
export const formatDateIST = formatDateLocalized
```

**Apply the same pattern to the sibling `formatISTDate` function** (used for `date_from`/`date_to` filter params) — same file, same `DEPLOY_LOCALE`/`DEPLOY_TIMEZONE` constants, renamed to `formatDateLocalizedForFilter` with a `formatISTDate` alias preserved.

**Add to `.env.local` (FRONTEND_02's file):**
```bash
NEXT_PUBLIC_DEPLOY_LOCALE=en-IN
NEXT_PUBLIC_DEPLOY_TIMEZONE=Asia/Kolkata
```
Defaults preserve exact current behavior for the existing Chennai deployment; a future non-Indian client deployment overrides both values.

**Note on call sites:** the deprecated aliases mean `AuditTimeline.tsx` (per `FRONTEND_SUPPLEMENT_05`'s own instruction to use `formatDateIST`) continues to work unchanged. Renaming every call site is not required by this amendment — only the underlying hardcoding is fixed.
